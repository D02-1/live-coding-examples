Was ist eine Single Page Application?

Single page applications sind einzelne seiten, die viele informationen liefern können, und spezifische inhalte aktualisieren, ohne die seite wechseln zu müssen.

Wenn wir beispielsweise auf GMAIL sind, und uns durch die emails und ordner klicken, sehen wir das sich ausser des hauptinhaltes nichts ändert, die seite läd nicht neu, nur spezifische inhalte werden aktualisiert.

SPA Architektur

Die Single Page Application ist eine Webanwendung oder "Webseite", die mit dem benutzer interagiert, indem sie die aktuelle seite dynamisch umschreibt, anstatt neue seiten vom server zu laden.

Dieser ansatz minimiert die unterbechung der benutzererfahrung zwischen den aufeinanderfolgenden seiten, so das die anwendung sich mehr wie eine desktop anwendung, also ein "computerprogramm" verhält.

Auf den meisten webseiten gibt es viele sich wiederholenden inhalte, einige davon bleiben immer gleich, egal wo der benutzer sich hinbewegt, zum beispiel der Header, der Footer, die Navigationsleiste, und so weiter. Allgemeinem bleiben viele elemente des layouts gleich.

Single Page Applications machen sich die wiederholung von elementen, sogenannten komponenten zu nutze.

Bei traditionellen Multi Page Applications, würden wir bei jeder seite die wir anfragen immer die ganze seite, mit allen inhalten geliefert kriegen.

Bei einer SPA (Single Page Application) hingegen, bekommt unser browser eine bauanleitung, wie er die seite zusammensetzen soll, und wo sich was wiederholt oder benutzt wird, und fügt es dann alles zusammen.

Die geschwingdigkeit der SPA kommt dadurch zu stande, das ab diesem zeitpunkt, nur noch einzelne informationen ausgetauscht werden müssen, aber keione webseite mit all ihren inhalten mehr komplett heruntergeladen werden muss, während bei der klassischen seite jetzt immernoch bei jeder anfrage, die seite mit allen inhalten neu geladen werden muss.#

Da bei SPA's die aufgabe des zusammensetzen der webseite auf seiten d es browsers liegt, kann der inhalt dynamisch neu geschrieben werden, anstelle komplett neu geladen werden zu müssen.

Anstelle das wir die seite immer neu laden, sehen wir also nur die neuen ersetzen inhalte.

Gerade im JavaScript bereich sind SPA's beliebt weil man schnell und einfach komplette webseiten, oder web-applikationen erschaffen und diese dann wie desktop apps nutzen kann.

Das Konzept MVC

MVC Steht für Model-View-Controller, eine spezielle struktur für web applikationen, in der die applikation in 3 komponenten eingeteilt wird, die voneinander getrennt entwicklt werden können.

  • Model: Die datebankanbidung, die mit dem Controller verbunden ist um daten auszutauschen
  • View: Die darstellung der daten auf der HTML seite, die über den controller zum benutzer kommen.
  • Controller: Die steuerung der applikation, hier sitzt die programmlogik die die daten und die sichtbare webseite verbindet.

Der vorteil dieser flexiblen art der programierung liegt in der einfachen wiederverwendbarkeit, und wartung. Im team ist es auch einfacher an MVC's zu arbeiten, wenn jeder seine bestimmten aufgabenbereiche hat. Allerdings ist das setup um eine MVC aufzubauen komplizierter und man ist möglicherweise abhängig von modulen anderer entwickler.

Was ist React?

React ist eine Javascript framework von FaceBook (Meta), genau wie bootstrap von Zurb Twitter kommt, hat facebook also mit React einen großen einfluss auf das gehabt, was wir als modernes web kennen.

React wurde entwickelt um interaktive benutzeroberflächen zu erstellen. Der hauptvorteil von React ist, das man sehr einfach verschiedene Views oder einzelne kleine teile, genannt komponenten, erstellen kann, die auf anfrage einfach geupdated werden können. Jeder komponent ist in der lage, seine eigenen daten und strukturen zu managen, weswegen man sie fantastisch zu komplexen benutzeroberflächen zusammenfügen kann.

Da die applikation dann aus einzelnen komponenten besteht, kann man sie voneinander getrennt updaten oder warten, ohne die gesuchte stelle auf tausendne zeilen html oder javascript suchen zu müssen.

React hat eine sogenannte state und prop verwaltung, das sind 2 wichtige konzepte von React.

  • Props sind eigenschaften die von aussen an komponenten übergeben werden um die funktionaltät zu erweitern, diese properties können innerhalb des komponenten verwendet werden. Das ist vergleichbar mit parametern die wir an eine funktion übergeben.
  • States sind eigenschaften die innerhalb einer komponente sind, um einfluss auf den zustand des views haben zu können. Diese states müssen auf spezielle art und weise gesetzt, genutzt und verändert werden, damit sie in echtzeit dynamisch verwendet werden können. Dies ist auch der grund, wieso React, React heisst, denn wenn sich ein state innerhalb einer komponente verändert, wird automatisch die renderfunktion aufgerufen, was dazu führt das die neuen daten direkt angezeigt werden, es wird auso auf einen state reagiert...reacted.

React arbeitet ausserdem mit sogenannten hooks, die die states die verschieden lebensphasen eines komponenten, wie zum beispiel den zeitpunkt des ladens des komponenten direkt managen können.

Fangen wir an mit React

Die einfachste möglichkeit am anfang mit react zu arbeiten, ist das terminal programm "create-react-app", das uns eine Rudimentäres react projekt erstellt, mit dem wir arbeiten können. Create-react-app können wir entweder mit npm installieren, oder die live-version nutzen, in dem wir npx nutzen.

Wir nutzen npx damit wir das auch mal sehen.

Wir erstellen einen ordner. gehen hinein und schreiben "npx create-react-app". und fügen am ende unseren projektnamen an. (npx create-react-app first app)

Wie wir sehen installiert es react und einige andere module... also warten wir mal ab.

In der zwischenzeit können wir mal in den browser, und uns ein chrome plugin namens "React Developer Tools" installieren, das wir in zukunft öfter mal benutzen werden um uns unsere komponenten im detail anzuschauen.

Wir geben bei google "React developer tools" ein, klicken das gesuchte ergebnis auf der chrome web store an, und installieren es in den browser.

React Developer Tools

create-react-app hat uns jetzt eine komplette applikation erstellt, die wir theoretisch einfach ausführen könnne, mit den befehlen die uns aus der package.json im terminal bereitgestellt wurden, aber schauen wir uns erst einaml an was wir jetzt haben.

Einige dateien und ordner wurden erstellt.

der node-modules ordner ist riesig, wie erwartet, hier ist alles drin, was wir zum starten von reat brauchen. Wenn wir unsere applikation erweitern, brauchen wir gegebenfalles noch andere module, aber für den anfang dürfte das reichen.

Außerdem haben wir die ordner public und src.

in public finden wir die haupt html datei, die die app ausführen wird, und alles was sonst noch zu der html datei gehört wie zum beispiel das favicon.

Im ordner src finden wir unser react projekt:

  • App.css beinhaltet unser css.
  • App.js ist unsere eigentliche applikation.
  • App.test.js beinhaltet tests für die erfolgreiche ausführung der applikation
  • index.css beinhaltet css für den body, und vordefinierte elemente.
  • index.js ist die dastei, die unsere app in der html datei aufruft.

Den rest können wir ignorieren, und uns mal die applikation anschauen.

Jetzt schreiben wir "npm run start" und sehen den live server von create-react-app laufen.

Wenn wir jetzt in der App.js änderungen machen, sehen wir, durch den live server, diese sofort.
